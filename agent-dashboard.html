The install pattern clients actually need.

A prospect does not need eight agents on day one. They need one narrow worker that proves the model, then a system that can expand safely.

01

Map the leak.

Find where time, follow-up, money, or reputation is leaking already: inbox, leads, delivery, admin, reporting.

02

Define the employee.

Give the agent a job, lane, memory, tools, cadence, forbidden actions, and a receipt format.

03

Launch narrow.

Run the manual loop first. Verify outputs. Only automate the parts that prove they behave.

Permissions are the product.

The magic is not the model. The magic is knowing what each agent may read, draft, touch, and never do.

Allowed by default

Read, summarize, draft.

Agents can inspect approved sources, draft emails/docs, create reports, and prepare the next move.

Approval gate

Send, publish, modify.

Outbound messages, live funnel edits, calendar changes, cron jobs, and client-facing actions wait for Jason.

Red lines

No secret spaghetti.

No exposed tokens, no personal credentials for client installs, no broad autonomy, no private-lane leakage.

Build the system beneath the work.
